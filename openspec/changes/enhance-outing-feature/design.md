## Context

当前游戏中"外出"入口藏在每日任务面板里，缺乏独立的交互流程：无条件检查、无动画反馈、无事件系统、无日记写入。本次变更将外出升级为一个完整的独立功能，涉及底部导航、角色状态机、事件数据层、日记面板四个模块，是一次跨多个代码区域的功能增强。

现有相关 `G` 状态字段：
- `energy`（≥50 可出门）、`mood`（≥40 可出门）
- `outingCooldown`（冷却时间戳，已有）
- `coins`（外出可获得）
- `diary`（日记数组，已预留但未使用）

## Goals / Non-Goals

**Goals:**
- 底部导航新增外出按钮，与喂食/换装平级
- 条件不足时展示撒娇拦截动画（随机对话 + 表情切换 2 秒）
- 条件满足时触发完整出行流程（扣能量→角色隐藏→20 秒倒计时→归来弹框）
- 归来弹框展示随机事件（常见纯文字 / 稀有图文）及偶发奖励
- 关闭弹框后自动将事件写入 `G.diary`（含日期）
- 小猪日记面板支持纯文字和图文混排两种渲染形式
- 所有提示文案从预设集合随机抽取

**Non-Goals:**
- 外出期间的后台离线处理（外出计时仅在页面开启时运行）
- 多角色或多地点外出
- 外出获得物品的背包/使用系统（本次仅展示，不实现消耗）
- 相册（`memories`）功能（另立变更）

## Decisions

### 1. 外出计时方案：前端 `setTimeout` vs 页面离开后持久化

| 方案 | 说明 | 结论 |
|------|------|------|
| `setTimeout` 20 秒（选用） | 简单，符合"页面开着玩"的轻量定位；20 秒体感短，离线处理意义不大 | ✅ 选用 |
| 存储出发时间戳，页面重载后补算 | 实现复杂，需处理倒计时 UI 恢复，超出本次范围 | ❌ 不选 |

保存 `G.outingStartedAt`（时间戳）以便未来可扩展离线处理，但本次不做恢复逻辑。

### 2. 撒娇拦截：覆盖层 vs 对话气泡

| 方案 | 说明 | 结论 |
|------|------|------|
| 临时全屏半透明覆盖层 + 居中对话框（选用） | 视觉清晰，与现有弹框风格一致 | ✅ 选用 |
| 复用现有 speech bubble | 无法同时切换角色表情 PNG | ❌ 不选 |

切换表情实现：临时将 `#char-img` 的 `src` 改为 `char_pouty.png`，2 秒后恢复，有 `onerror` emoji 回退。

### 3. 倒计时展示：CSS 动画 vs JS `setInterval`

用 `setInterval`（1 秒刷新）更新倒计时数字，CSS 负责样式；避免纯 CSS 动画难以精确同步逻辑。

### 4. 事件数据结构

```js
OUTING_EVENTS = [
  { id, rarity: 'common', text: '…' },               // 常见：纯文字
  { id, rarity: 'rare',   text: '…', img: 'path' },  // 稀有：图文
]
```

稀有事件图片路径由用户提供（`assets/outing/`），缺失时降级为纯文字展示（`onerror`）。

### 5. 日记条目结构

```js
// G.diary 条目
{ date: '2026/4/3', text: '…', img?: 'path', rarity: 'common'|'rare' }
```

日记面板根据 `img` 字段是否存在决定使用纯文字还是图文布局。

### 6. 奖励概率

- 金币奖励：80% 概率，随机 5–20 枚
- 物品奖励：20% 概率（本次仅弹框展示名称，不实现背包）

## Risks / Trade-offs

- **计时器被页面隐藏节流** → 20 秒体感可能变长；可接受，属于浏览器限制
- **`char_pouty.png` 尚未提供** → `onerror` 回退到 🥺 emoji，不影响功能
- **稀有事件图片缺失** → `onerror` 降级纯文字，日记条目不存 `img` 字段
- **`game.js` 行数增长** → 新增约 150–200 行；仍在单文件合理范围内

## Migration Plan

纯增量变更，无破坏性改动：
1. `G.diary` 字段已预留（数组），直接 push 新条目即可；老存档读取时为空数组，正常展示"暂无日记"
2. `G.outingStartedAt` 为新字段，老存档不存在时视为 `null`（无进行中的外出）
3. 底部导航新增按钮不影响现有喂食/换装逻辑

## Open Questions

- 撒娇对话集合的具体文案由用户最终确认（设计文档提供示例占位）
- 稀有事件图片（外出照片）的提供方式与路径规范待用户确认
- 外出冷却时间是否复用现有 `outingCooldown`（建议复用，本次实现时确认）
